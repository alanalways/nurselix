# nurslix 戰略重定位 Design Spec

> 2026-05-17 · Alan + Claude（brainstorming session）
> 狀態：草案，等使用者 review

## 1. 為什麼要重定位

### 客觀現況（2026-05-16 snapshot）

- **使用者**：21 個註冊，**過去 7 天 0 活動**
- **題庫使用率**：14,323 題只有 489 題（3.4%）被答過
- **付費**：0 個付費訂單
- **回報品質**：11 筆使用者回報全 IN_REVIEW，最有價值用戶 `aas253000` 抱怨「解析不清楚」多次，第二個用戶 `a0928383872` 試一次就跑了
- **Feedback**：0 筆評分留言（剛裝彈窗）

### 工程現況

- **規模**：~36k LOC、40 個 DB models、104 個 API endpoint、12 個 cron、8 個 AI agent
- **品質**：0 真實 TS error、1 個 TODO、health score 100/100 — 工程紀律極佳
- **問題**：**8-10 個獨立子系統，1 個人維護，21 個用戶** — 典型「功能膨脹、價值稀釋」

### 戰略診斷

> 「nurslix 像是為了證明 AI 工程能跑通而存在的 demo，不是為了賣給之後 1000 個人而存在的產品。」

工程不是瓶頸；**定位過度發散 + 對誰賣不夠清楚** 才是。

## 2. 重定位後的核心 pitch

來自使用者親口的兩句話：

> **「台灣不缺好的護理師，缺的是看得懂英文的護理師。」**
> **「大家都想要更好的薪水跟環境，只要訓練英文能力這些應該不是問題。」**

合併為正式 pitch：

> **台灣護理師加薪 3-5 倍的方法只有一個：去美國。
> 去美國的關鍵不是技術，是英文。
> nurslix 給你 14,323 題 NCLEX 練到看懂英文，順便考過。**

### Framing 轉換

| 舊 framing | 新 framing |
|---|---|
| NCLEX 題庫 SaaS | **護理師加薪訓練營**（透過考 NCLEX 出國） |
| 對手：UWorld、Kaplan | 對手：UWorld + Google 翻譯（笨拙組合） |
| 賣點：題量、AI | 賣點：**英中對照解析 + 英文閱讀力同步成長** |
| 定價：年費 NT$3,000-6,000 是天花板 | **月費 NT$390-500 是合理區間**（職涯投資價） |
| 目標：通過 NCLEX | 目標：**通過 NCLEX 並開始用得上工作英文** |

## 3. 目標客群（聚焦）

**ICP（理想用戶）**：
- 18-30 歲、台灣護理系學生 / 已執業護理師
- 正在準備或考慮考 NCLEX-RN
- 英文閱讀能力中等（看得懂單字但讀不通整題）
- 月可支配收入 NT$3,000-10,000 用於進修

**6 個月目標**：
- 100-200 個付費用戶
- 月費 NT$390-500
- 月營收 NT$50,000-100,000（達成題目「月收 5-30 萬」的下緣）

## 4. 採用方案：「我跟你一起考 NCLEX」內容創作者路線

### 4.1 核心 framing

使用者本人（Alan）是執業護理師、正在/計劃 3-6 個月內考 NCLEX。他**就是 nurslix 的目標用戶**。

→ 把「準備 NCLEX 的過程」當行銷內容主軸。

### 4.2 為什麼這個方案最強

1. ✅ 使用者**就是**ICP — 不用「假裝」
2. ✅ 內容寫真實體驗，**0 編造成本**
3. ✅ 考過後 = 最強商品背書
4. ✅ 對手永遠不會做這個（UWorld 沒有「華人創辦人也在考」這故事）
5. ✅ 利用使用者既有的「臨床經驗 + 工程能力」雙重稀缺性

### 4.3 3 個月 milestone

| 月 | 主要動作 | 數字目標 |
|---|---|---|
| **M1（2026-05 → 2026-06）** | 個人天天用 + 招呼 3-5 個身邊朋友 alpha + 開始 Threads/IG 每日紀錄 | 5-10 個活躍 alpha + 30 篇社群貼文 |
| **M2（2026-06 → 2026-07）** | 「準備 NCLEX 第 X 天」系列 + 收 testimonial + 修產品 bug | 200+ Threads 追蹤、3-5 篇 testimonial |
| **M3（2026-07 → 2026-08）** | 考過 NCLEX（或考完）→ case study → 上線付費版 | 首 10 個付費用戶 |

### 4.4 內容策略（每週）

| 平台 | 頻率 | 內容類型 | 範例主題 |
|---|---|---|---|
| **Threads** | 每天 1 篇 | 「準備 NCLEX 第 X 天」+ 今天卡的英文題 | 「today's NCLEX 卡在 anticipate which order FIRST 這個句構」 |
| **Instagram 限時** | 每天 1 則 | 用 nurslix 練習截圖、學習日常 | session 結果、答錯題 + 中文解釋 |
| **Instagram 貼文** | 每週 2-3 篇 | 整理過的學習筆記（圖文） | 「DKA 急救順序中英對照」「ABG 三步驟」 |
| **YouTube Shorts**（可選） | 每週 2 支 | 60 秒「我每天怎麼用 nurslix 自學 NCLEX」 | 實機操作影片 |

### 4.5 產品調整方向

#### 4.5.1 砍掉 / 冰凍

| 模組 | 動作 | 理由 |
|---|---|---|
| **TOEIC 模組**（`phase16_toeic_audio.sql`、`app/api/admin/tts/`、`/toeic/*` 頁面） | **冰凍**：保留 DB schema 和資料；前端 sidebar 移除 TOEIC 連結、`/toeic/*` 路由 redirect 到 `/nclex`；audit-worker 不再產 TOEIC 題；admin 後台保留入口（你之後想恢復可隨時） | TOEIC 跟 NCLEX 學生無直接相關 |
| **VocabSession 獨立模組** | 整合進 NCLEX 題目頁（hover 即查），下架獨立入口 | 護理英文單字該在題目情境中學，不是獨立 app |
| **24/7 audit-worker** | 改為每週末跑一次 | 14k 題已審完，持續跑是燒錢 |
| **Marketing AI 全自動** | 已改成半自動（commit `4a0e30b`） | 完成 |

#### 4.5.2 強化（核心戰場）

| 功能 | 動作 | 為何重要 |
|---|---|---|
| **題目英中對照解析** | 確認 14k 題每題都有：英文題幹 + 中文翻譯 + 英中對照 rationale | 這是新定位的**唯一賣點** |
| **單字 hover 即查** | 在 NCLEX session 頁，每個英文字 hover 顯示中文 + 詞性 + NCLEX 常考用法 | 重用 `VocabularyWord` table，做出新 UI |
| **Hermes 英文卡點分析** | Hermes prompt 加入：「分析使用者答錯是英文問題還是知識問題」 | 強化個人化價值 |
| **題目「英文難度」標籤** | 在 Question table 加 `englishDifficulty: EASY/MED/HARD`，給使用者選 | 讓學生從簡單英文題開始建立信心 |

#### 4.5.3 補（重要新功能）

| 功能 | 動作 |
|---|---|
| **落地頁重寫**（`/`） | 用「台灣護理師加薪 3-5 倍」當 hero、附使用者自己的「我也在考 NCLEX」真實故事 |
| **「英文閱讀力」儀表板** | 在 user dashboard 顯示「你能看懂 X% 的 NCLEX 題目」（以答對率推算） |
| **Alpha 用戶招募頁** | 簡單 form：留 email → 送 PRO 60 天免費 → 換 1 句證言 |

### 4.6 定價策略

| 方案 | 月費 | 內容 | 預估比例 |
|---|---|---|---|
| **FREE** | NT$0 | 每天 5 題、無英中對照 | 80% 用戶 |
| **BASIC** | NT$390 | 無限題目、英中對照解析、Hermes | 15% 用戶 |
| **PRO** | NT$590 | + 弱點分析 / 模考 / 考前衝刺包 | 5% 用戶 |

**6 個月目標**：100 BASIC + 20 PRO = NT$50,800/月

### 4.7 風險與緩衝

| 風險 | 緩衝 |
|---|---|
| 使用者考試延後或放棄 → 沒 case study | spec 寫 3-6 個月彈性，不綁死時間點 |
| Alpha 用戶不付費 → 商業模式不驗證 | 跟前 5 個 alpha 用戶事先約定「考過後願意付月費」 |
| 14k 題的中文解析品質不夠 → 留不住人 | 用使用者自己練題發現的「解析不清楚」當每週修補清單 |
| 砍 TOEIC 模組後有人抱怨 | 先確認 21 個用戶中無 TOEIC 活躍使用者再砍（看 `VocabSession` 表） |

## 5. 不做的事（紅線）

- ❌ **不做台灣國考**（換題庫成本太高）
- ❌ **不做香港/星馬市場**（語言版本暴增、無法分身）
- ❌ **不做完全自動發文到 Threads/IG**（封號風險，且失去「真實」這個賣點）
- ❌ **不做 KOL 業配**（成本高、ROI 不穩定，留待 100 付費用戶後再考慮）
- ❌ **不做進階：發音矯正、臨床對話模擬**（YAGNI，等 200 付費用戶後再說）

7. **冰凍**：將 TOEIC、Marketing 全自動 cron disable
